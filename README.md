# 五子棋游戏 (Gomoku)

一个基于 C++ 实现的命令行五子棋游戏，支持人机对战。

## 项目简介

本项目是一个五子棋游戏实现，支持人机对战。AI 采用 Minimax 算法配合 Alpha-Beta 剪枝，通过棋型分析评估局面。游戏在命令行界面运行，玩家执黑先行，AI 执白，先连成五子者获胜。

## 功能特性

- **人机对战**：玩家执黑，AI 执白，轮流落子
- **AI 引擎**：基于 Minimax + Alpha-Beta 剪枝的搜索算法
- **棋型评估**：支持五连、活四、死四、活三、死三、活二、死二、活一、死一等棋型识别与评分
- **候选走法优化**：仅搜索已有棋子周围 2 格内的空位，排序后取 Top 15 加速剪枝
- **胜负判定**：自动检测五子连珠
- **坐标输入**：通过标准输入进行落子
- **棋盘显示**：带坐标轴和网格的棋盘显示
  - `●` 表示黑子
  - `○` 表示白子
  - `·` 表示空位交叉点

## 编译运行

### 环境要求

- C++17 或更高版本编译器
- Windows/Linux/macOS

### 编译命令

```bash
g++ -std=c++17 -o gomoku main.cpp
```

### 运行游戏

```bash
./gomoku
```

## 游戏说明

1. 游戏开始后，黑方（玩家）先行
2. 玩家输入坐标 `x y` 进行落子（坐标范围：1-38）
3. AI 自动计算并落子
4. 系统会自动判断胜负和平局
5. 任意一方连成五子即获胜
6. 棋盘填满未分胜负则为平局

## 技术细节

### 类设计

- `GoGame`: 游戏核心类，包含棋盘状态和游戏逻辑
  - `set(x, y, player)`: 落子
  - `undo(x, y)`: 撤销落子
  - `validPosition(x, y)`: 检查坐标合法性
  - `Win(x, y, player)`: 检查是否获胜
  - `GameOver()`: 检查游戏是否结束
  - `getPlayer(x, y)`: 获取指定位置棋子颜色
  - `show()`: 显示棋盘

- `Minimax`: AI 引擎类，实现 Minimax 搜索与棋型评估
  - `getBestMove(g)`: 获取当前局面最优走法
  - `setDepth(depth)`: 设置搜索深度
  - `getNodesSearched()`: 获取已搜索节点数
  - `getSelf()`: 获取 AI 执子颜色
  - `alphaBeta(g, depth, alpha, beta, isMaximizing)`: Alpha-Beta 剪枝搜索
  - `evaluate(g)`: 全局局面评估
  - `evaluatePosition(g, x, y, p)`: 单点威胁评估
  - `analyzeDirection(g, x, y, dx, dy, p)`: 方向棋型分析
  - `getCandidateMoves(g)`: 生成候选走法
  - `checkWin(g, p)`: 检查指定玩家是否获胜

### AI 算法

- **搜索算法**：Minimax + Alpha-Beta 剪枝
- **搜索深度**：默认 4 层（可配置）
- **评估方式**：遍历所有空位，分别计算己方和对方的棋型威胁分值
- **防守权重**：默认 1.2，可调节 AI 攻守倾向
- **候选走法**：收集已有棋子周围 2 格内空位，按综合评分排序取 Top 15
